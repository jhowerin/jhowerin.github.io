IEX Group API's

Details on the IEX API's can be found HERE. The API supports GET requests with various options and the data returned is JSONP.
    The GET request syntax
  1. Prefix = https://api.iextrading.com/1.0
  2. Endpoints = {tops || market || stats}
  3. Parameters
  4. Remember the ? to indicate this is a GET request
The complete GET request is Prefix + Endpoint + Parameters
Let's look at the tops endpoint. The tops endpoint has numerous attributes and parameters. At a minimum, the user must provide a stock symbol. For example, with the symbol aapl (Apple Computers), the GET request will look like the following:https://api.iextrading.com/1.0/tops/?symbols=aapl. You can filter the results by adding in a filter parameter that you can use to determine which JSON key/value pairs you want to see. So, if you want to only see the last traded stock price, you can use filter=lastSalePrice. The complete URL would be https://api.iextrading.com/1.0/tops/?symbols=aapl&filter=lastSalePrice. With this approach, you can access all of the publically available stock information provided by IEX.
GET Request Attribute Table
Endpoint Parameters Note
top symbol The stock symbol for the real-time quote
top filter Filter the results of those you want to see. Key point is that you need to know the JSON key/value pairs in order to filter on the keys. To see all the key/value pairs, issue a GET request without filtering, learn the key/value pairs, and then issue another GET request including any desired filters.
market format Overall market data
stats records IEX trading macro stats. There are many other attributes for stats documented at the IEX website.

Make your own GET request

From what you've learned so far, you are ready to craft your own GET request messages. Remember the formula is Prefix + Endpoints + Parameters. Try the examples provided as a start. Enter a complete GET request and see what happens ! Please note that due to cross-origin issues, the GET request will be prepended with a URL to a CORS proxy.
